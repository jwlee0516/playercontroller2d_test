Hollow Knight Feel

+ Movement Model
+ Tiny State Machine
+ Forgiving Input Tech

Forgiveness: coyote time, jump buffer, early-jump-cut, variable jump height
Snappy horizontal control: fast accel/decel on ground, softer in air, no ice
Gravity shaping: higher gravity on the way down + fas-fall clamp (weight)
Dash: short, crisp, temporarily ignores gravity, locked duration, cooldown, preserves facing
Wall interactions: sticky wall slide with controlled speed, wall jump with brief input lock
Micro-feel: small “apex” hang, friction when no input, consistent max speeds, input buffering for dash/jump

Tuning guide (this is where the “feel” lives)
If it doesn’t feel HK-ish yet, tweak in this order:

1. Run snap
	* Increase groundAccel / groundDecel until it feels instant but not jittery
	* Keep airAccel lower than ground, but not too low (HK still has good air control)
	* If direction changes feel sluggish: raise turnBoost

2. Jump forgiveness
	* coyoteTime ~ 0.08–0.12
	* jumpBufferTime ~ 0.08–0.12
	* variable jump: jumpCutMultiplier 0.45–0.65

3. Weight
	* Raise baseGravityScale until falls feel “heavy”
	* Then raise fallGravityMultiplier until descent feels decisive
	* Clamp: maxFallSpeed prevents infinite terminal velocity and keeps consistent landing timing

4. Dash crispness
	* dashDuration 0.14–0.18
	* dashSpeed 16–20
	* dashCooldown 0.22–0.32
	+ HK dash feels like it temporarily suspends physics; that’s why gravity is off and velocity is forced.

5. Wall slide
	* wallStickTime tiny (0.05–0.10) helps the “catch” before sliding
	* wallSlideSpeed 3–5 depending on scale

Things to Consider
	- Nail slash recoil / pogo (down-slash bounce is huge for feel)
	- Ledge coyote + step-up (micro ledges)
	- Slope handling (HK has lots of curved terrain)
	- Attack cancel windows / dash cancel rules
	- Animation hooks (events for run, land, jump, dash, wall)
